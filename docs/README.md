## Documentation Overview

### ONBOARDING.md
- **Purpose**: Guide new developers from zero to productive
- **Length**: ~5 days of structured learning
- **Format**: Checklist-based
- **Audience**: Brand new team members

### DEVELOPER_GUIDE.md
- **Purpose**: Day-to-day development reference
- **Content**: Workflows, debugging, common tasks
- **Format**: How-to guides and examples
- **Audience**: All developers (reference material)

### ARCHITECTURE.md
- **Purpose**: Deep technical understanding
- **Content**: System design, data flow, decisions
- **Format**: Detailed explanations and diagrams
- **Audience**: Technical leads, architects, experienced devs

### QUICK_REFERENCE.md
- **Purpose**: Fast lookup while coding
- **Content**: Code snippets, commands, patterns
- **Format**: Tables and code blocks
- **Audience**: All developers (quick lookup)

## Contributing to Documentation

Found something unclear or missing?

1. **Minor fixes**: Edit the file directly and commit
2. **Major changes**: Discuss with team first
3. **New sections**: Update this index file too

### Documentation Standards

- Use clear, simple language
- Include code examples
- Keep it up-to-date with code changes
- Link between documents when relevant
- Use markdown formatting consistently
